# Ctrip IMExperience Field Mapping

## Filters to session-list fields

- 客人来源 `Trip` -> `productChannel: "trip"`
- 客人来源 `CTrip` -> expected `productChannel: "ctrip"`
- 客人来源 `汇总` -> expected `productChannel: "aggregate"`
- 咨询场景 `汇总` -> `consultationScene: "aggregate"`
- 咨询场景 `售前` -> expected `consultationScene: "presale"`
- 咨询场景 `售后` -> expected `consultationScene: "postsale"`
- 历史咨询日期 `昨天` -> `startDate/endDate: 2026-06-17` in this capture
- 分页 -> `pageNo/pageSize`
- 排序 -> `orderColumn/orderType`

## Customer-service account to session-list fields

From `getEmployeeDimMetricDetailsV3` response:

- `tableDataItemList[].dimMap.vendor_account_id` -> `searchMap.vendor_account_id`
- `tableDataItemList[].dimMap.vendor_account_name` -> `searchMap.vendor_account_name`

## Session detail

From `getSessionDimMetricDetailsV3` response:

- `tableDataItemList[].dimMap.session_id` -> detail URL `queryMessages?...&sessionId=<session_id>`
- Detail page API request body -> `{"sessionId":"<session_id>"}` for `16037/getMessagesBySession`

## Dynamic fields

- `session_id`: per conversation, do not hard-code.
- `pageNo`: generated by pagination.
- `startDate/endDate`: generated from selected date range.
- Cookie/session headers: read from `ctrip-cli-sessions`, do not hard-code.
