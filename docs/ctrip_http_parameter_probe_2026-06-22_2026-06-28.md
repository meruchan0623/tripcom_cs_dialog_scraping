## 参数范围建议

| 参数 | 已验证范围/值 | 建议值 | 说明 |
|---|---:|---:|---|
| `startDate` / `endDate` | `YYYY-MM-DD` | 显式传入 | 避免默认昨天和跨时区歧义 |
| 员工列表 `pageSize` | `1` 到 `500` 成功 | `1000` | 跟随全局默认；员工接口本次单独实测到 500 |
| 会话列表 `pageSize` | `1` 到 `1000` 成功 | `1000` | 本次已验证的最大稳定值，请求数最少 |
| `pageNo` | `1` 到 `ceil(totalNum/pageSize)`；越界返回空 | 从 1 递增到空页或尾页 | 不需要预先知道最大页 |
| `maxPages` | 由 CLI 控制 | `50` | 对当前最大 212 会话/客服足够 |
| `productChannel` | 本次 `aggregate` | 按业务筛选 | `trip`、`ctrip` 需另行实测 |
| `consultationScene` | 本次 `aggregate` | 按业务筛选 | `presale`、`postsale` 需另行实测 |
| `butype` | `品类活动` | 保持配置 | 与当前页面业务一致 |

## 阻碍与注意事项

- 纯 `requests` 直连在当前环境仍可能第一跳返回空体 `HTTP 403`；本次参数探测使用的是已登录页面上下文请求。
- 本文档验证的是客服会话列表接口，不包含 1383 条会话逐条详情消息正文导出。
- 如需导出全部详情消息，预计至少需要按会话数发起约 1383 次详情接口请求，应单独设置请求预算、节流和失败恢复策略。
- 不要将真实 cookie、ticket 或 token 写入文档、日志或提交。
